Практичне заняття: Реалізація шаблону Singleton у програмі
Мета заняття
Навчитися проектувати та реалізовувати шаблон Singleton для забезпечення одного екземпляра класу.

1. Що таке Singleton?
Singleton — це шаблон проєктування, який гарантує, що клас має лише один екземпляр, та надає глобальну точку доступу до нього.

Де використовується: логування, конфігурація, підключення до БД, кешування.

2. Структура проєкту
Файл	Призначення
singleton.py	Простий Singleton
singleton(2).py	Singleton з лінивою ініціалізацією
singleton(3).py	Потокобезпечний Singleton
singleton(4).py	ConfigManager (конфігурація)
singleton(5).py	Logger (логування)
singleton(6).py	DatabaseConnection (БД)
golovna_proga.py	Головна програма для демонстрації
3. Реалізовані види Singleton
Простий Singleton – створює один екземпляр через метод __new__.

Лінива ініціалізація – створює екземпляр тільки при першому зверненні.

Потокобезпечний Singleton – використовує блокування для безпечної роботи в багатопотокових програмах.

ConfigManager – керує налаштуваннями програми, зберігає їх у JSON-файл.

Logger – записує логи у файл з часовими мітками.

DatabaseConnection – керує єдиним підключенням до бази даних.

4. Демонстрація роботи
Головна програма викликає всі реалізації:

Простий Singleton показує однакові значення лічильника

ConfigManager завантажує конфігурацію

Logger записує повідомлення

Результат: всі Singleton працюють коректно.

5. Висновки
У ході роботи реалізовано різні види Singleton на Python. Патерн корисний для створення єдиних точок доступу до ресурсів, але потребує уваги до потокобезпеки. Найкраще підходить для логерів, конфігурацій та підключень до БД.
