<img width="1915" height="603" alt="image" src="https://github.com/user-attachments/assets/730c6393-5106-45ee-a424-48b27353f89a" />В ході практичного заняття було розроблено веб-додаток з використанням фреймворку Flask. Для порівняння також створено альтернативну версію на чистому Python без зовнішніх бібліотек.

Для проєктування додатку було створено схему бази даних з двома основними моделями: User та Post. Модель User містить поля id, username, email, password_hash, created_at та is_active. Модель Post включає id, title, content, created_at, updated_at та зовнішній ключ user_id. Зв'язок між таблицями один до багатьох.

Структура проєкту включає основний файл app.py, models.py для роботи з базою даних, forms.py для валідації форм, security.py для засобів безпеки, test_app.py для тестування та deploy.py для розгортання. Також створено шаблони HTML: base.html, index.html, register.html, login.html, dashboard.html, profile.html, create_post.html та сторінки помилок 404 і 500.

Міграцію бази даних виконано за допомогою SQLAlchemy. Використано SQLite для розробки. Взаємодія з базою даних реалізована через ORM, що дозволяє виконувати операції створення, читання, оновлення та видалення даних.

Контролери обробляють маршрути: головна сторінка, реєстрація, вхід, панель управління, створення постів, профіль користувача та вихід. Кожен контролер перевіряє стан сесії користувача та перенаправляє на сторінку входу при необхідності. Відображення даних реалізовано за допомогою шаблонізатора Jinja2.

Бізнес-логіка включає реєстрацію нових користувачів з перевіркою унікальності імені та email, автентифікацію з перевіркою пароля, створення постів з валідацією довжини полів, відображення постів на панелі управління в зворотному хронологічному порядку. Форми валідуються на стороні сервера з перевіркою обов'язкових полів, мінімальної довжини, формату email та співпадіння паролів.

Засоби безпеки реалізовано на кількох рівнях. Для захисту від SQL-ін'єкцій використано ORM SQLAlchemy, який автоматично параметризує запити. Від XSS захищає функція екранування HTML-тегів та блокування небезпечних патернів. CSRF-захист забезпечує Flask-WTF з генерацією унікальних токенів для кожної форми. Паролі хешуються за допомогою bcrypt з сіллю. Сесії налаштовано з HttpOnly, Secure та SameSite атрибутами. Проведено валідацію всіх вхідних даних з очищенням потенційно небезпечного вмісту.

Відладка виконувалась за допомогою вбудованого дебагера Flask. Тестування проведено з використанням pytest. Написано тести для перевірки головної сторінки, реєстрації, входу та виходу, захищених маршрутів, створення постів та захисту від XSS-атак. Усі тести успішно пройдено.

Документація включає опис структури проєкту, інструкцію з встановлення та запуску, опис API маршрутів та функціоналу. Для деплойменту підготовлено скрипт deploy.py, який перевіряє середовище, встановлює залежності, створює базу даних, запускає тести та налаштовує production-конфігурацію. Рекомендовано використовувати gunicorn як WSGI-сервер та nginx як зворотний проксі. Для production середовища рекомендується замінити SQLite на PostgreSQL.

Основні досягнуті результати: повністю функціональний веб-додаток з автентифікацією, управлінням постами та захистом від основних веб-вразливостей. Додаток може бути розгорнутий на сервері для публічного доступу. Альтернативна версія на чистому Python демонструє можливість створення веб-додатку без використання сторонніх фреймворків, але з меншою продуктивністю та гіршою масштабованістю.
